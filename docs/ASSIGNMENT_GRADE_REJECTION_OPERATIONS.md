# 과제 성적: 반려·코멘트 운영 가이드 (Handong Judge)

## 목적

자동 채점(DomJudge)이 **AC**를 준 경우에도, 수업·과제 요구(예: 특정 함수 사용)를 위반한 제출을 **강사·튜터가 반려**하고, **학생이 사유를 확인**한 뒤 **재제출**할 수 있게 하는 절차를 정리합니다.  
(적용 범위: **과제(assignment)**. 코딩 테스트(퀴즈)는 본 문서의 반려·코멘트 UI 대상이 아닙니다.)

## 튜터·강사 절차

1. **성적 관리**에서 분반·과제를 선택해 성적 테이블을 연다.  
   - 수업 전체 보기 / 특정 과제 보기 모두, **과제** 문제 셀에서 상태 배지(예: 정답, 미제출) **오른쪽**의 `</>` 버튼을 사용한다.
2. `</>`를 누르면 **코드 확인 · 코멘트 · 반려** 모달이 열린다.  
   - 제출이 있으면 **제출 코드**가 로드된다.  
   - **학생에게 보이는 코멘트**에 사유·재제출 요청·기한 등을 적는다.
3. **반려 처리**를 체크하면:  
   - 이 문제 **점수는 0점**으로 저장된다.  
   - **반려**를 해제하고 저장하면, 수동 점수가 비워져 **자동 채점(정답이면 배점)** 기준으로 다시 반영된다(수업 정책에 따라 재제출 후 다시 검토).
4. 반려 시에는 **코멘트 필수**로 두었습니다(저장 시 빈 코멘트면 안내).

## 학생 쪽 동작

1. 해당 **과제 문제 풀이 페이지** 상단에 **반려** 태그와 **강사 코멘트**가 표시된다.  
2. **문제 목록** 모달에서도 반려된 문제에 **· 반려** 표시가 붙는다.  
3. 재제출 가능 여부는 과제 **마감·재오픈 정책**을 수업에서 따로 안내한다(시스템은 알림 메일을 자동 발송하지 않을 수 있음).

## 운영 시 권장

- 수업 공지·이메일 등으로 **재제출 마감**, **위반 유형**, **문의 담당**을 명시한다.  
- 반려는 **문제 단위**로 저장되므로, 여러 문제 위반 시 문제마다 코멘트를 남기는 것이 학생 이해에 유리하다.  
- 성적 합산은 성적 테이블의 **문제별 점수 합**에 맞춰진다(반려 시 해당 칸 0점).

## 기술 메모 (참고)

- API: `POST /api/sections/{sectionId}/assignments/{assignmentId}/grades` body에 `rejected: true`, `score: 0`, `comment` 등.  
- DB `grades` 테이블에 `rejected` 컬럼이 추가된다(`ddl-auto: update` 환경 기준).
